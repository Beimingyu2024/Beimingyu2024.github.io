一、前言

这篇文档算是我的网站搭建完成后的首发，以此纪念个人网站的成立和复盘搭建的过程，方便后面还需要在创建新的网站少走一点弯路。我并非计算机专业出身，但是工作后对计算机逐渐开始产生了兴趣，同时近期也想开始不断尝试记录，可以不需要过度公开，因此想到创建自己的网站。搭建的这第一个网站参考了很多大佬们的教程，以下相关内容仅用作个人学习使用，而我也会不断学习精进，在未来某一天能够自己DIY代码，从头到尾的设计自己的个性化网站。

2.3 安装并配置Node.js

(1) 安装Node.js-它让JavaScript成为与PHP、Python、Perl、Ruby等服务端语言平起平坐的脚本语言。

主要教程：https://blog.csdn.net/weixin_52799373/article/details/123840137

辅助教程：https://blog.csdn.net/antma/article/details/86104068

直达下载：http://nodejs.cn/download/

Node.js安装时遇到的问题及解决方案：https://blog.csdn.net/weixin_52799373/article/details/125718587?spm=1001.2014.3001.5502

注意：建议使用安装程序进行安装。安装时，请勾选Add to PATH选项。

(2) 环境变量配置

  • 配置全局安装的模块路径和缓存路径

    • 在node.js根目录,创建node_global，node_cache文件夹

    • 管理员身份- 打开CMD，配置路径：

      注意一定要管理员身份运行CMD！！！路径改成你自己的路径，不要无脑复制！！！

      1. 在开始菜单搜索栏中输入cmd;
      2. 在搜索结果上右键点击“命令提示符”;
      3. 选择“以管理员身份运行”。

  • 配置环境变量

    • 找到电脑环境变量位置：右键此电脑——属性——高级系统设置——高级——环境变量

    • 在“环境变量”下创建 NODE_HOME 变量，变量值为node.js地址

  • 在“系统变量”中选择“Path”修改和添加属性

    • node.js的路径修改为“%NODE_HOME%”

    • 添加node_global，node_cache

(3) 测试

  • 查看node的版本：node -v

    查看npm的版本(新版的node安装自带安装npm)： npm -v

  • 使用快捷部署hexo的方式——test_node

    在存放install.sh脚本的目录中,右键打开git bash然后运行以下命令:

    source install.sh test_node

    • 出现了版本号和成功的提示信息代表配置成功

    • 如果并未发现有版本号的出现,而是红色的错误信息,可能需要手动配置node.js的环境变量,或重新安装node.js并注意勾选Add to PATH。

  • 全局安装最常用的 express 模块 进行测试

    • npm install express -g

生成SSH Keys + 绑定Github

  • 采用快捷部署方式：https://hexoscript.gitbook.io/hexo-script/v/1.3-1/shi-yong-bu-zhou/yi-jian-sheng-cheng-ssh-mi-yao

  • 常规方式：

    • （1）生成SSH： ssh-keygen -t rsa -C "你的邮箱地址"

      如果生成有错误，找到C/Users/Administrator下一长串的数字，删除后重新执行生成SSH的命令即可。

    • （2）用生成的SSH密钥绑定Github

      参考教程：https://zhuanlan.zhihu.com/p/103391101

    • （3）测试ssh是否绑定成功：ssh -T git@github.com
